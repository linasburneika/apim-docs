## Introduction

The [Reports Config widget](https://www.npmjs.com/package/@itwin/reports-config-widget-react) is a UI that simplifies the creation, modification, and extraction of Reports within the Reporting Platform.
This tutorial will guide you through some basic steps and provide references to set up a viewer application for Reports Config Widget.

By the end of this walk-through, you will be able to have an iTwin viewer with quick access to all your reports.

</Introduction>

## Permissions and Scopes

Do not forget to put these values within the `IMJS_AUTH_CLIENT_SCOPES` environment variable.

In addition, users must have the `imodels_read` and `imodels_write` [permissions](/apis/insights/operations/create-mapping/#authorization) assigned at either the Project or iModel level. Further instruction on how to create roles and assign permissions can be found in the [iTwin Platform Access Control API documentation](/apis/access-control-v2/tutorials/).

</TutorialSection>

## Adding the Reports Config Widget

The first step required by the widget is to initialize it before making use of the provider. This will initialize the widgetâ€™s localization.

Next, you must add the `ReportsConfigProvider` to the list of `uiProviders` within the viewer component.
More details on adding `uiProviders` can be shown in this [tutorial](/tutorials/itwin-viewer-hello-world/#2-your-first-ui-widget).

Congratulations, you should now have the Reports Config widget within your iTwin Viewer application.
